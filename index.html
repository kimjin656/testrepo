📊 RDB 관계 가이드

JSONPlaceholder 리소스를 활용한 관계형 데이터베이스 입문

1. 전체 관계도 (ERD 개념)

시스템의 중심인 Users를 기준으로 모든 데이터가 연결됩니다.

Users (1)
Posts (N)
Comments (N)
Users (1)
Albums (N)
Photos (N)

2. 1:N (One-to-Many) 관계 설명

Case A

Users ↔ Posts

한 명의 사용자는 여러 개의 게시글을 작성할 수 있습니다.

연결 고리: Posts.userId
Case B

Posts ↔ Comments

하나의 게시물에는 여러 개의 댓글이 달릴 수 있습니다.

연결 고리: Comments.postId
Case C

Albums ↔ Photos

하나의 앨범 안에는 여러 장의 사진이 포함됩니다.

연결 고리: Photos.albumId

3. 핵심 용어 정리

  • PK (Primary Key, 기본 키): 각 데이터를 유일하게 식별하는 신분증 (예: user.id)
  • FK (Foreign Key, 외래 키): 다른 테이블과 관계를 맺기 위해 가져온 상대방의 PK (예: post.userId)

4. 데이터 조회 방식 비교

특정 게시물(ID: 1)의 모든 댓글을 가져오는 방법

SQL (RDB)

SELECT * FROM comments 
WHERE postId = 1;

REST API

fetch('.../posts/1/comments')
  .then(res => res.json())
  .then(console.log);

5. 왜 데이터를 쪼개나요? (정규화)

1. 용량 최적화: 중복 데이터를 방지하여 저장 공간을 아낍니다.

2. 데이터 무결성: 사용자 정보 수정 시 한 곳만 고치면 연결된 모든 데이터에 반영됩니다.
